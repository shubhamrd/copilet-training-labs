# Exercise: Develop Python Unit Tests with GitHub Copilot

> Time to Complete: ~45 minutes

GitHub Copilot is a fantastic tool for taking the tedious work out of writing unit tests, documentation, and CI/CD pipelines.

In this comprehensive lab, we will build a Python Calculator class, automatically generate a test suite, practice Test-Driven Development (TDD), and finally, generate a GitHub Actions pipeline to automate our testing.

---

## 🛡️ Free Tier Best Practices

If you are using the GitHub Copilot Free Tier, you have a monthly limit of 50 Chat messages and 2,000 Completions.

- **Conserve Chat:** We will use standard code "Completions" (gray ghost text generated by typing comments) for most of this lab. This saves your highly limited Chat quota for complex tasks!
- **Close Tabs:** Copilot reads your open tabs to build context. Close tabs you aren't using to improve accuracy and save tokens.

## Step 1: Set up the Workspace

Let's create a blank workspace and our core application file.

1. Open github.com and sign in with your credentials. Create a new blank repository named `python-test-app`, select **Add a README** during repository creation, 
2. Select code -> Codespace tab --> Create codespace on mail to open the repository in a Codespace.
3. Open your terminal (`Ctrl + ~` or `Cmd + ~`).

4. Create a new file named `calculator.py` from left side file management.

---

## Step 2: Create the Application Code (Completions)

We need some business logic to test. We will use a "Zero-Shot Prompt" comment to generate our code without using the Chat window.

1. Open `calculator.py`.

2. Type the following exact comment at the top of the file and press **Enter**:

    ```python
    # Create a Calculator class with add, subtract, multiply, and divide methods.
    # The divide method should raise a ValueError if dividing by zero.
    ```

3. Wait 1–2 seconds. Copilot will suggest the code in gray text.

4. Press **Tab** to accept GitHub Copilot's suggestions.

    > **Note:** If it generates one line at a time, keep pressing **Enter** and **Tab** until the class is complete.

### Expected Output (`calculator.py`)

Save the file once it looks similar to this:

```python
class Calculator:
    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

    def divide(self, a, b):
        if b == 0:
            raise ValueError("Cannot divide by zero")
        return a / b
```

---

## Step 3: Generate Unit Tests (Copilot Chat)

Now we will use our first Chat request. We will use the dedicated `/tests` command to generate a complete testing suite instantly.

1. Highlight the entire `Calculator` class inside `calculator.py` using your mouse.

2. Open the **Copilot Chat** panel on the left sidebar (the chat bubble icon).

3. Type the following prompt into the chat box and press **Enter**:

    ```
    /tests generate unit tests for this class using Python's built-in unittest library
    ```

4. Copilot will analyze your highlighted code and generate a comprehensive test suite.

---

## Step 4: Save and Run the Tests

1. In the Copilot Chat window, hover over the top right of the generated code block. Click the **"Insert into New File"** icon (or click `...` → **Insert into New File**).

2. Save this new file in your project folder as `test_calculator.py`.

### Expected Output (`test_calculator.py`)

```python
import unittest
from calculator import Calculator

class TestCalculator(unittest.TestCase):
    def setUp(self):
        self.calc = Calculator()

    def test_add(self):
        self.assertEqual(self.calc.add(2, 3), 5)
        self.assertEqual(self.calc.add(-1, 1), 0)

    def test_subtract(self):
        self.assertEqual(self.calc.subtract(5, 3), 2)

    def test_multiply(self):
        self.assertEqual(self.calc.multiply(3, 4), 12)

    def test_divide(self):
        self.assertEqual(self.calc.divide(10, 2), 5)
        with self.assertRaises(ValueError):
            self.calc.divide(10, 0)

if __name__ == '__main__':
    unittest.main()
```

3. Open your terminal in VS Code.

4. Run the unit tests using Python's built-in test runner:

    ```bash
    python -m unittest test_calculator.py
    ```

5. You should see an `OK` message indicating all 4 tests passed.

## Step 5: Ask Copilot for Edge Cases (Copilot Chat)

A great way to use AI is to ask it to think of testing scenarios you might have missed.

1. Go back to the **Copilot Chat** panel.

2. Ensure `test_calculator.py` is open and active in your editor.

3. Ask Copilot the following question:

    ```
    Are there any edge cases we missed testing in the Calculator class? Provide the code to add those tests.
    ```

4. Copilot will suggest testing things like extremely large numbers, floating-point arithmetic, or negative numbers.

5. Use the **"Insert at Cursor"** button in chat to seamlessly add one of Copilot's suggested edge-case tests into your `test_calculator.py` file.

6. Run the tests in your terminal again to verify the new edge cases pass:

    ```bash
    python -m unittest test_calculator.py
    ```

## Step 7: Test-Driven Development (TDD) Loop

In a real enterprise environment, developers often write tests before they write the feature. Let's practice TDD using Copilot.

### Part A: Write the Failing Test

1. Open `test_calculator.py`.

2. Scroll to the bottom of your test methods (before the `if __name__ == '__main__':` block).

3. Type the following comment and press **Enter**:

    ```python
    # Test the modulo method with standard and negative numbers
    ```

4. Press **Tab** to accept the generated test method. Expected file content:

    ```python
    import unittest
    from calculator import Calculator

    class TestCalculator(unittest.TestCase):
        def setUp(self):
            self.calc = Calculator()

        def test_add(self):
            self.assertEqual(self.calc.add(2, 3), 5)
            self.assertEqual(self.calc.add(-1, 1), 0)
            self.assertEqual(self.calc.add(0, 0), 0)

        def test_subtract(self):
            self.assertEqual(self.calc.subtract(5, 3), 2)
            self.assertEqual(self.calc.subtract(0, 5), -5)

        def test_multiply(self):
            self.assertEqual(self.calc.multiply(4, 3), 12)
            self.assertEqual(self.calc.multiply(-2, 3), -6)
            self.assertEqual(self.calc.multiply(0, 5), 0)

        def test_divide(self):
            self.assertEqual(self.calc.divide(10, 2), 5)
            self.assertAlmostEqual(self.calc.divide(3, 2), 1.5)

        def test_divide_by_zero_raises(self):
            with self.assertRaises(ValueError):
                self.calc.divide(5, 0)

        def test_float_and_negative_edge_cases(self):
            self.assertAlmostEqual(self.calc.add(1.1, 2.2), 3.3000000000000003)
            # Fix: Use 10**18 (integer) instead of 1e18 (float) to prevent precision errors
            self.assertEqual(self.calc.subtract(10**18, 1), 999999999999999999)
            self.assertEqual(self.calc.multiply(-1e6, -2), 2000000)
            self.assertEqual(self.calc.divide(-10, 2), -5)
            self.assertAlmostEqual(self.calc.divide(1, 3), 0.3333333333333333)

        def test_type_errors_for_invalid_inputs(self):
            with self.assertRaises(TypeError):
                self.calc.add("1", 2)
            with self.assertRaises(TypeError):
                self.calc.subtract(1, "2")
            with self.assertRaises(TypeError):
                self.calc.multiply("a", "b")
            with self.assertRaises(TypeError):
                self.calc.divide("10", "2")

        # Test the modulo method with standard and negative numbers
        def test_modulo(self):
            self.assertEqual(self.calc.modulo(10, 3), 1)
            self.assertEqual(self.calc.modulo(-10, 3), 2)
            self.assertEqual(self.calc.modulo(10, -3), -2)
            self.assertEqual(self.calc.modulo(-10, -3), -1)

    if __name__ == "__main__":
        unittest.main()
    ```

5. Run your tests in the terminal. The test will **fail** because the `modulo` method doesn't exist yet! This is exactly what we want.

    ```bash
    python -m unittest test_calculator.py
    ```

### Part B: Write the Feature

1. Open `calculator.py`.
2. Add a new line at the bottom of the `Calculator` class.
3. Type `def modulo` and wait.
4. Copilot knows about the failing test because the file is open in the next tab! It will automatically suggest the correct mathematical logic (`return a % b`):

    ```python
    def modulo(self, a, b):
        """Return the remainder of a divided by b. Raise ValueError if b is zero."""
        if b == 0:
            raise ValueError("Cannot modulo by zero")
        return a % b
    ```

5. Press **Tab** to accept.

6. Run your tests again. They should now pass!

    ```bash
    python -m unittest test_calculator.py
    ```

## Step 8: Generate a CI/CD Pipeline

Finally, let's automate our unit tests so they run every time someone pushes code to our repository using GitHub Actions.

1. In the root of your project folder, create a new file structure: `.github/workflows/python-app.yml`.

    You can do this in the terminal using:

    ```bash
    mkdir -p .github/workflows && touch .github/workflows/python-app.yml
    ```

2. Open the empty `python-app.yml` file.

3. Type the following comment at the top of the file and press **Enter**:

    ```yaml
    # GitHub Actions workflow to run Python unit tests on every push to main
    ```

4. Wait for Copilot to generate the YAML pipeline.

5. Press **Tab** to accept the completion.

### Expected Output (`python-app.yml`)

```yaml
name: Python application
on:
  push:
    branches: [ main ]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python 3.x
        uses: actions/setup-python@v2
        with:
          python-version: '3.x'
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
      - name: Run tests
        run: |
          python -m unittest test_calculator.py
```
