# Building & Adapting Python Web Services

## Hands-On Lab with GitHub Copilot

In this lab, you will:

- Integrate Copilot into VS Code.
- Build a basic REST API using Flask.
- Use AI to translate the API into FastAPI.
- Test your endpoints without leaving your editor.

---

## Lab Prerequisites

Before we begin, ensure you have:

- Python 3 installed on your machine.
- Visual Studio Code (VS Code) installed.
- An active GitHub account with a Copilot subscription.
- An internet connection to download Python packages.

---

## Step 1: Connect Copilot to VS Code

1. Open Visual Studio Code.
2. Click the Extensions icon on the left menu (or press `Ctrl+Shift+X`).
3. Search for **"GitHub Copilot"** and click **Install**.
4. A prompt will appear in the bottom right: click **Sign in to GitHub**.
5. Your browser will open. Authorize VS Code to access your account.
6. **Verify:** Look for the Copilot icon (the robot face) in the bottom-right status bar of VS Code.

---

## Step 2: Initialize Your Workspace

1. Open your VS Code Terminal (**View > Terminal**).

2. Run these commands to create your project folder:

    ```bash
    mkdir copilot-python-lab
    cd copilot-python-lab
    ```

3. Create and activate a virtual environment:

    ```bash
    python -m venv venv
    ```

    > **Windows:** `venv\Scripts\activate` | **Mac/Linux:** `source venv/bin/activate`

4. Install the required frameworks:

    ```bash
    pip install flask fastapi uvicorn requests
    ```

---

## Step 3: Prompting for a Flask API

1. Create a new file named `app_flask.py`.

2. Type the following exact comment at the top of the file and press **Enter**:

    ```python
    # Create a Flask REST API for a To-Do list.
    # Use an in-memory list called 'todos'.
    # Include a GET route at /todos to retrieve all items.
    # Include a POST route at /todos to add a new item.
    # Listen on port 5000.
    ```

3. Wait 2–3 seconds for Copilot to generate the gray ghost text, then press **Tab** to accept.

### Expected Flask Output

Copilot should generate something very similar to this. Save your file!

```python
from flask import Flask, request, jsonify

app = Flask(__name__)
todos = []

@app.route('/todos', methods=['GET'])
def get_todos():
    return jsonify(todos)

@app.route('/todos', methods=['POST'])
def add_todo():
    data = request.get_json()
    if 'item' not in data:
        return jsonify({"error": "Missing 'item' in request body"}), 400
    todo_item = data['item']
    todos.append(todo_item)
    return jsonify({"message": "Todo item added", "item": todo_item}), 201

if __name__ == '__main__':
    app.run(port=5000)
```

## Step 4: Test the API using Copilot

> Since terminal commands like `curl` differ on Windows/Mac, let's use Copilot to write a test script!

### First, start your Flask Server

1. In your active VS Code terminal, run the Flask app:

    ```bash
    python app_flask.py
    ```

2. Leave this terminal running! You should see terminal output indicating that the server is listening on `http://127.0.0.1:5000`.

### Next, create the Test Script

1. Open a new, second terminal tab in VS Code (click the **+** icon in the terminal panel).

2. Change to your project directory in the new terminal:

    ```bash
    cd copilot-python-lab
    ```

3. Create a file named `test_api.py`.

4. Use Copilot to write the test by typing this comment:

    ```python
    # Write a Python script using 'requests' to send a POST request to http://127.0.0.1:5000/todos
    # with a JSON payload {"item": "Learn Copilot"}, then print the response.
    ```

### Expected Test Code

```python
import requests

url = "http://127.0.0.1:5000/todos"
payload = {"item": "Learn Copilot"}
response = requests.post(url, json=payload)

print(response.status_code)
print(response.json())
```

5. Press **Tab** to accept the code and save the file.

6. Run the test script in your second terminal. Make sure you are in the `copilot-python-lab` directory.

    ```bash
    python test_api.py
    ```

### Expected Output

```
201
{'message': 'Todo item added', 'item': 'Learn Copilot'}
```

## Step 5: Switch to FastAPI

Let's use Copilot Chat to translate our code to a modern framework!

1. Create a new file named `app_fastapi.py`.
2. Keep `app_flask.py` open in the tab next to it (Copilot needs to "read" it).
3. In `app_fastapi.py`, press `Ctrl + I` (Windows) or `Cmd + I` (Mac) to open **Inline Chat**.
4. Type: `Translate the To-Do list API from app_flask.py into FastAPI.`
5. Click **Accept / Keep**.

### Expected FastAPI Output

Notice how Copilot automatically switched the decorators and added Pydantic validation!

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()
todos = []

class Todo(BaseModel):
    task: str

@app.get("/todos")
def get_todos():
    return todos

@app.post("/todos", status_code=201)
def add_todo(todo: Todo):
    todos.append(todo.dict())
    return todo
```

---

## Step 6: Test the FastAPI Docs

FastAPI automatically generates a testing UI. You don't even need a script!

1. Stop your old Flask server in the first terminal (`Ctrl + C`).

2. Run your new FastAPI server:

    ```bash
    uvicorn app_fastapi:app --reload --port 8000
    ```

3. Open your web browser and go to: <http://127.0.0.1:8000/docs>

4. Expand the **POST** route and click **Try it out**.

> [!IMPORTANT]
> **AI Generation Check:** Look at your generated `app_fastapi.py` code. What did Copilot name the variable inside your `class Todo(BaseModel):`? Because AI generation varies, it might be named `task`, `item`, or `title`.
>
> You must match your JSON payload to that exact word!
>
> For example, if it generated `title: str`, enter `{"title": "Learn Copilot"}` in the testing UI.

5. Click **Execute!**
