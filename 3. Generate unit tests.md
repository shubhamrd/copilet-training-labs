# Exercise: Develop Unit Tests using GitHub Copilot

The large language models behind GitHub Copilot are trained on a variety of code testing frameworks and scenarios. GitHub Copilot is a great tool for generating test cases, test methods, test assertions and mocks, and test data. In this exercise, you use GitHub Copilot to accelerate the development of unit tests for a C# application.

---

## Exercise Scenario

You're a developer working in the IT department of your local community. The backend systems that support the public library were lost in a fire. Your team needs to develop a temporary solution to help the library staff manage their operations until the system can be replaced. Your team chose GitHub Copilot to accelerate the development process.

You have an initial version of the library application that includes a unit test project named `UnitTests`. You need to accelerate the development of additional unit tests using GitHub Copilot.

---

If you don't have a GitHub account, you can sign up for a free individual account and use a GitHub Copilot Free plan. If you have access to a Pro, Business, or Enterprise subscription, you can use your existing subscription.

Your lab environment must include the following:

- .NET SDK 9.0 or later
- Visual Studio Code with the C# Dev Kit extension


## Step 1: Set up the Library Application in Visual Studio Code

You need to download the existing application, extract the code files, and then open the solution in Visual Studio Code.

1. Open a browser window in your lab environment.
2. To download a zip file containing the library application, navigate to: **GitHub Copilot lab - develop unit tests**. The zip file is named `AZ2007LabAppM4.zip`.
3. Extract the files from the `AZ2007LabAppM4.zip` file.
4. Copy the extracted `AccelerateDevGHCopilot` folder to a location that's easy to access, such as your Windows Desktop folder.
5. Open Visual Studio Code. On the **File** menu, select **Open Folder** and select the `AccelerateDevGHCopilot` folder.
6. In the Visual Studio Code **SOLUTION EXPLORER** view, verify the solution structure contains:

    - `src\Library.ApplicationCore`
    - `src\Library.Console`
    - `src\Library.Infrastructure`
    - `tests\UnitTests`

7. Ensure that the solution builds successfully. Right-click `AccelerateDevGHCopilot` in the Solution Explorer, and select **Build**.

    > **Note:** You'll see some Warnings, but there shouldn't be any Errors.

## Step 3: Extend the UnitTests Project (Agent Mode)

You can use the Chat view's **Agent** mode when you have a specific task in mind and want to enable Copilot to autonomously edit your code and create files.

> [!NOTE]
> Agent mode is only available in Visual Studio Code.

1. In the Chat view, select the **Set Mode** drop-down button, and then select **Agent**.

2. To start an automated task that creates a test class, enter the following prompt:

    ```
    Add `Infrastructure\JsonLoanRepository` folders to the UnitTests project. Create a class file named `GetLoan.cs` in the `JsonLoanRepository` folder and create a stub class named `GetLoan`. Add a reference to the Library.Infrastructure project inside UnitTests.csproj.
    ```

3. The agent displays status messages as it completes the requested tasks. If the agent pauses and asks you for permission to run a make directory command in the terminal, select **Continue**.

4. In the Chat view, to accept all changes, select **Keep**.

5. In the Solution Explorer, verify the new folder structure: `UnitTests\Infrastructure\JsonLoanRepository\GetLoan.cs`.

## Step 4: Use Edit Mode to Create Unit Tests for the GetLoan Method

1. In the Chat view, select the **Set Mode** drop-down button, and then select **Edit**.

2. Open the `GetLoan.cs` test file and select the `GetLoan` class.

3. Add the following files to the Chat context:

    - `LoanService.cs`
    - `JsonData.cs`
    - `JsonLoanRepository.cs`
    - `ReturnLoan.cs`
    - `LoanFactory.cs`

4. Enter the following prompt:

    ```
    #codebase Create fields and a class constructor for the `GetLoan.cs` file. The class will be used to create unit tests for the GetLoan method in the `JsonLoanRepository.cs` file. Create the following private readonly fields: `_mockLoanRepository`, `_jsonLoanRepository`, `_configuration`, and `_jsonData`. Instantiate the fields in the `GetLoanTest` constructor. Use `ConfigurationBuilder` to create a `_configuration` object that can be used to instantiate the JsonData object.
    ```

5. In the Chat view, select **Keep** to accept the generated constructor.

6. Ensure your `GetLoan.cs` looks similar to this (resolve any namespace or `using` conflicts manually if needed):

    ```csharp
    using NSubstitute;
    using Library.ApplicationCore;
    using Library.ApplicationCore.Entities;
    using Library.Infrastructure.Data;
    using Microsoft.Extensions.Configuration;

    namespace UnitTests.Infrastructure.JsonLoanRepositoryTests;

    public class GetLoanTest
    {
        private readonly ILoanRepository _mockLoanRepository;
        private readonly JsonLoanRepository _jsonLoanRepository;
        private readonly IConfiguration _configuration;
        private readonly JsonData _jsonData;

        public GetLoanTest()
        {
            _mockLoanRepository = Substitute.For<ILoanRepository>();
            _configuration = new ConfigurationBuilder().Build();
            _jsonData = new JsonData(_configuration);
            _jsonLoanRepository = new JsonLoanRepository(_jsonData);
        }
    }
    ```

7. Add `Loans.json` to your Chat context. Select the contents of the `GetLoan.cs` file, and enter the following prompt in the Chat view:

    ```
    @workspace Update the selection to include a unit test for the `JsonLoanRepository.GetLoan` method. The unit test should test the case where a loan ID is found in the data. Use `_mockLoanRepository` to arrange the expected return loan. Use `_jsonLoanRepository` to return an actual loan. Asserts should verify that the return loan ID matches the expected loan ID. Use a loan ID that exists in the `Loans.json` file.
    ```

8. Select **Keep** to accept the updates. Copilot will generate a `[Fact]` test verifying the success condition.

9. Build the solution to ensure there are no errors.

### Autocomplete the Failure Case

1. Create a blank line after the `GetLoan_ReturnsLoanWhenLoanIdIsFound` method you just created.
2. Wait a moment, and accept the inline autocompletion suggestions (using **Tab**) to create a new test method for the case where the loan ID isn't found.
3. Ensure the mocked failure test utilizes an ID that does not exist (e.g., `999`) and asserts a `null` return.

---

## Step 5: Update the Project Configuration

Notice that the unit tests require access to the JSON data files. You need to update the `UnitTests.csproj` file to include these files in the test project so they copy over during the build.

1. Open `UnitTests.csproj`.
2. Add the following XML snippet inside the `<Project>` tags:

    ```xml
    <ItemGroup>
        <None Include="..\..\src\Library.Console\Json\**\*">
            <Link>Json\%(RecursiveDir)%(FileName)%(Extension)</Link>
            <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
        </None>
    </ItemGroup>
    ```

---

## Step 6: Run the Unit Tests

1. Build the solution and ensure that there are no errors.
2. Open Visual Studio Code's **Test Explorer** view (select the beaker-shaped icon on the left-side Activity bar).
3. Expand `UnitTests` and locate `GetLoanTest`.
4. Run the `JsonLoanRepository.GetLoan: Returns loan when loan ID is found` test case.
5. You should see a green checkmark indicating the test passed.
6. Use the code Editor to run the `JsonLoanRepository.GetLoan: Returns null when loan ID is not found` test case by clicking the green play button to the left of the test method in the code. Ensure it passes.
