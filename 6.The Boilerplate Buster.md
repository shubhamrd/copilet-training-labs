# Lab: The Boilerplate Buster

## Scenario

Because the local Windows VM lacks a Docker installation, students will spin up a GitHub Codespace. Within this cloud environment, they will use Copilot to rapidly generate a complete, production-ready project skeleton for a Dockerized Python Web Server (FastAPI).

---

## Step 1: Provisioning the GitHub Codespace

1. Navigate to GitHub.com and create a new, empty public repository (e.g., `copilot-boilerplate-lab`).
2. On your new repository page, click the green `<>` Code button, select the Codespaces tab, and click **Create codespace on main**.
3. Wait for the web-based VS Code environment to initialize. (This cloud environment automatically includes Docker and Python).
4. Verify the GitHub Copilot extension is active in the bottom-right corner of the Codespace browser window. If you are prompted to sign in, use your GitHub credentials.

---

## Step 2: Defining the Scope with Copilot Chat

1. Open the GitHub Copilot Chat window within your Codespace.
2. Switch the chat mode to **Plan**.
3. Prompt Copilot: "Design a complete boilerplate structure for a lightweight Python web server using FastAPI. It must include a structured directory layout (`app/` and `tests/`), a `requirements.txt`, and a `Dockerfile`."

---

## Step 3: Generating the Skeleton Structure

1. Review the generated plan in the chat.
2. Click **Start Implementation** (or manually switch to Agent Mode) to instruct Copilot to execute the plan. If asked, select simple application and create without any extra parameters.
3. Allow Copilot to autonomously create the file tree. Verify in the Explorer pane that `app/main.py`, `requirements.txt`, and `Dockerfile` are populated correctly.

---

## Step 4: Generating the Testing Harness

1. Open `app/main.py` and highlight all the code.
2. Press `Ctrl + I` (`Cmd + I` on Mac) to open the Inline Chat.
3. Prompt: "Create a comprehensive unit testing suite using pytest for these endpoints. Put it in `tests/test_main.py`."
4. Review the generated tests and click **Accept** to apply the changes.

---

## Step 5: Validation Run (Docker in Codespaces)

1. Open the integrated terminal in the Codespace (`Ctrl + `` ` or `Cmd + `` `).
2. Press `Ctrl + I` (`Cmd + I` on Mac) in the terminal and prompt: "How do I build and run this Dockerfile locally, mapping port 8000?"
3. Execute the suggested commands (typically something like):
   ```bash
   docker build -t fastapi-app . && docker run -d -p 8000:8000 fastapi-app
   ```
4. Navigate to the **Ports** tab next to the terminal panel and click the forwarded URL (or click the "Open in Browser" toast notification) to verify the FastAPI server is running.

---

## Step 6: Environment Cleanup (Important)

1. Close the Codespace browser tab.
2. Navigate to `github.com/codespaces` in a new browser tab.
3. Locate the active Codespace used for this lab, click the three dots (`...`) on the right side of the row, and select **Delete**.
4. **Crucial context**: Always delete unused Codespaces to conserve your free monthly compute quota provided by GitHub.
